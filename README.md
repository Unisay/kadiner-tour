# Kadiner Tour — виртуальный тур по квартире (Pannellum)

360°-тур по квартире на Kadiner Straße. Интерфейс и названия комнат — **на немецком**
(тур показывают арендаторам/гостям). Движок — [Pannellum](https://pannellum.org/),
завендорен локально в `vendor/` (офлайн, без CDN).

## Структура

```
assets/        оригиналы с камеры (22 JPG 11904×5952 + 22 DNG-сырца) — НЕ в git, ~9 ГБ
images/        веб-панорамы scene01..scene22.jpg (4096px, ~600 КБ) — то, что отдаётся
thumbs/        превью для просмотра/контактных листов — НЕ в git
vendor/        pannellum.js + pannellum.css (2.5.6)
scenes.json    ← РЕДАКТИРУЕМЫЙ источник: имена комнат, ракурсы, хотспоты
tour.json      ← генерируется из scenes.json (руками не править)
index.html     вьюер: меню комнат + режим расстановки хотспотов
scripts/       build-images.sh, build-tour.py
justfile       команды
images/mapping.tsv   scene01 → исходный файл (провенанс)
```

## Запуск

```bash
just serve     # http://localhost:8000/   (редактор: .../?edit)
```
`file://` не годится — `index.html` грузит `tour.json` через fetch, нужен http-сервер.

## Как это устроено

- `scenes.json` — человекочитаемый источник. `python3 scripts/build-images.sh` тянет
  оригиналы из `assets/`, делает веб-версии и превью. `build-tour.py` компилирует
  `scenes.json` → `tour.json` (формат Pannellum). `just build` делает оба шага.
- Навигация: **меню «☰ Räume»** (слева) всегда доступно — тур проходим сразу, даже
  без пространственных переходов.

## Расстановка переходов между комнатами (хотспотов)

Координаты двери (yaw/pitch) на искажённой панораме на глаз не угадать, поэтому есть
встроенный редактор:

1. Открой `http://localhost:8000/?edit` → кнопка **«✎ Правка»** (внизу справа).
2. Покрути сцену так, чтобы центральный крестик **＋** смотрел точно на дверь/проём.
3. Выбери целевую комнату в списке → **«＋ Хотспот-переход здесь»**
   (или **«ℹ Инфо-хотспот»** для подписи).
4. Скопируй накопленный JSON из текстового поля.
5. Перенеси записи в `scenes.json` в `hotspots[]` нужной сцены (поле `from` подсказывает,
   в какую сцену класть), затем `just tour` и перезагрузи страницу.

Формат хотспота в `scenes.json`:
```json
{ "type": "scene", "sceneId": "scene04", "yaw": 120, "pitch": -5,
  "text": "Zum Flur", "targetYaw": -30 }     // targetYaw — необяз., куда смотреть после перехода
{ "type": "info",  "yaw": 40,  "pitch": 0, "text": "Balkon 6 m²" }
```

## Правка названий комнат / стартового ракурса

Имена и `yaw`/`pitch` в `scenes.json` — **черновые** (расставлены по виду превью).
Поправь `title`, выставь `yaw`/`pitch` (удобный начальный ракурс смотри в readout
редактора), смени `firstScene` при желании → `just tour`.

## Направление взгляда при переходах

Стрелки-переходы по умолчанию **сохраняют направление взгляда** (`targetYaw:
"sameAzimuth"` проставляется компилятором всем scene-хотспотам): кликнул — новая сцена
открывается на том же `yaw`, что и текущий вид. Навигация через меню «☰ Räume» открывает
сцену на её собственном `yaw`/`pitch`.

**Закреплённый угол.** Чтобы сцена всегда открывалась под одним и тем же углом (как ни
зайди), пометь её в `scenes.json`:
```json
{ "id": "scene06", "title": "Bad — Übersicht", "yaw": 12, "pitch": -4, "hfov": 100,
  "fixedView": true, "hotspots": [ ... ] }
```
`build-tour.py` тогда проставит этот `yaw`/`pitch`/`hfov` как `targetYaw/targetPitch/
targetHfov` всем входящим стрелкам (перекрывая `sameAzimuth`). В редакторе это делают
кнопки **«📌 Зафиксировать угол сцены»** (захватывает текущий вид → `action: "fix"`) и
**«Открепить угол»** (`action: "unfix"`); закреплённая сцена помечена 📌 в readout.

**Угол прихода (зависит от того, откуда пришёл).** Чтобы одна и та же сцена открывалась
под разным углом в зависимости от входной стрелки, задаётся `targetYaw`/`targetPitch` на
КОНКРЕТНОЙ входящей стрелке в `scenes.json` (а не на сцене; сцена при этом не должна быть
`fixedView`):
```json
{ "type": "scene", "sceneId": "scene13", "yaw": 30, "pitch": -3,
  "text": "Zum Flur", "targetYaw": 120, "targetPitch": -2 }
```
В редакторе: встань в целевой сцене, наведи нужный вид, выбери источник в «🎯 Угол прихода
СЮДА из …» (список — только сцены со стрелкой сюда) и нажми «Запомнить угол прихода»
(`action: "arrive"` с `from`/`to`/`targetYaw`/`targetPitch`). Стрелки без своего угла прихода
остаются на `sameAzimuth`.

## Публикация

Тур опубликован на **GitHub Pages**: https://cv.functional.work/kadiner-tour/
(репозиторий https://github.com/Unisay/kadiner-tour, ветка `main`, корень).

Обновить онлайн после правок в `?edit` (которые пишут `scenes.json` + пересобирают
`tour.json` локально):
```
just publish              # git add -A + commit + push  -> Pages пересоберётся за ~1 мин
just publish "msg"        # со своим сообщением коммита
```
В публикацию не идут `assets/` (оригиналы 9 ГБ), `thumbs/`, `backups/` (см. `.gitignore`).
Публичный сайт грузит только `tour.json` (read-only); backend/`serve.py` нужен лишь локально.

### Альтернатива — ekdos
```
/ekdos:publish index.html   # либо опубликовать всю папку — см. /ekdos:publish
```
Для веб-публикации в git нужны только `index.html`, `vendor/`, `images/`, `tour.json`
(остальное в `.gitignore`).
